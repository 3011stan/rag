# Business Rules

This document records behavior that should remain explicit as the RAG project evolves.

## API Access

- `POST /rag/ask` is public in the portfolio demo.
- `POST /rag/ingest` is protected in production.
- `POST /admin/seed-demo` is protected.
- Debug and destructive operations must stay protected.

## Temporary Token Access

Temporary tokens may grant limited access for demo or short-lived workflows.

Allowed temporary token scopes:

- Upload documents when explicitly scoped for upload.
- List/read allowed resources when explicitly scoped for that use case.

Temporary tokens must not allow:

- Delete operations.
- Debug/admin operations.
- Demo seeding.
- Secret inspection.
- Broad administrative access.

## Metadata

- Curation metadata is optional.
- Technical metadata is generated by loaders.
- Curation metadata must not overwrite technical/reserved fields.
- Curation metadata may overwrite semantic/editorial fields when explicitly provided.
- Existing documents without the new metadata schema must continue to work.

## Retrieval

- Retrieval should not use hard metadata filters by default.
- Metadata should initially be used as preferences for soft boosting.
- Relevant chunks from other layers should still be able to cooperate in an answer.
- Layered retrieval is a future option if context composition needs stronger control.

## Public Demo Safety

- Public upload should remain disabled or protected.
- The demo should prefer preloaded, curated documents.
- Private StanOS notes must not enter the public/demo corpus by accident.
